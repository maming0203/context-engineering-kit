# Compliance Rule Matcher（合规规则匹配）

## When to Use

当用户请求涉及以下内容时使用此技能：

- 对文档、代码或流程进行合规性检查
- 按规则集逐条审查目标内容
- 生成合规状态报告
- 识别违规项并提供整改建议

**触发关键词：** 合规、检查、审查、compliance、审计

## Steps

1. **加载规则集** — 确定适用的合规框架和具体规则列表。
   - 用户应指定框架名称（如 GDPR、SOX、内部制度等）
   - 如未指定，询问用户确认适用框架

2. **解析目标内容** — 读取并理解被审查的对象：
   - 文档类型（政策文件、代码、流程说明等）
   - 内容范围和边界

3. **逐条匹配** — 对每条规则：
   - 在目标内容中查找相关证据
   - 判定状态：pass / fail / warning / not-applicable
   - 记录判定依据（evidence）

4. **汇总统计** — 计算各状态的规则数量。

5. **生成建议** — 对 fail 和 warning 项：
   - 说明违规原因
   - 提供具体整改建议
   - 标注优先级（高/中/低）

6. **输出报告** — 按配方定义的 output_format 生成结构化报告。

## Pitfalls

- ⚠️ **不提供法律意见**：此技能仅做规则匹配，最终合规判定需法律专业人士确认。
- ⚠️ **规则时效性**：合规规则可能已更新，需确认使用的是最新版本。
- ⚠️ **上下文依赖**：某些规则的判定需要业务上下文，不得脱离上下文机械匹配。
- ⚠️ **遗漏规则**：必须逐条检查所有规则，不得跳过或合并。
- ⚠️ **证据不足**：如果无法找到充分证据，应标记为 warning 而非 pass。

## Verification

- [ ] 所有规则均已检查，无遗漏
- [ ] 每条判定都有明确的证据支持
- [ ] fail/warning 项均有整改建议
- [ ] 统计数据与逐条结果一致
- [ ] 未修改被审查的原始内容
