# Code Review Skill

## When to Use

Use when the user asks to review code.

## Steps

1. Check for bugs and logic errors
2. Evaluate code style and readability
3. Identify performance issues
4. Suggest improvements

## Pitfalls

- ⚠️ Do not assume the code's purpose — ask if unclear.
- ⚠️ Do not rewrite code without explanation.

## Verification

- [ ] All major issues identified
- [ ] Suggestions are actionable
